## Key Dependencies

- `docutils>=0.21,<0.23` - Node types and SphinxTranslator base class; TypstTranslator (~2700 lines) extends this
- `typst>=0.15.0,<0.16` - Typst compiler wrapper (`typst-py` package); used in `pdf.py` for PDF compilation via `typst.compile()`

**Development (optional, `extras = ["dev"]`):**
- types-docutils 0.21-0.22 - Type stubs for docutils
- twine 5.0+ - Package upload utility (PyPI)
- build 1.0+ - PEP 517 build frontend (used by GitHub Actions in release workflow)
- pypdf 6.14-6.x - PDF manipulation (imported but not shown active in current codebase; reserved)

## Configuration

**Environment:**
- Development: `uv sync --extra dev` creates isolated venv
- CI/CD: GitHub Actions (`ci.yml`, `release.yml`, `docs.yml`, `drift.yml`) manages matrix testing, lint, type checks, coverage, building, and publishing
- Worktree isolation: Per-worktree `uv sync` required; editable install (`__editable__.typsphinx-*.pth`) points to worktree source

**Build:**
- `pyproject.toml`: PEP 621 format; version, metadata, entry points, tool config all defined here
- `tox.ini`: Test/lint/type/docs environment definitions; runner = `uv-venv-lock-runner`
- Tool config embedded in pyproject.toml:
  - `[tool.black]`: line-length 88, target-version [py312, py313]
  - `[tool.ruff]`: line-length 88, target-version py312, specific rule ignores (N802 for docutils visitor PascalCase)
  - `[tool.mypy]`: python_version 3.12, permissive settings (disallow_untyped_defs=false), module overrides for typsphinx
  - `[tool.pytest.ini_options]`: testpaths=["tests"], markers (slow, integration), filterwarnings (error::DeprecationWarning, error::PendingDeprecationWarning)

## Platform Requirements

**Development:**
- Linux (primary CI), Windows, macOS (tested in CI matrix)
- Python 3.12 or 3.13
- `uv` installed globally or via `astral-sh/setup-uv@v7` GitHub Action
- Git (for worktree isolation and pre-commit hooks)

**Production:**
- Deployment target: PyPI (Python Package Index)
- Distribution: sdist (.tar.gz) + wheel (.whl) via GitHub Actions release workflow
- Installation: `pip install typsphinx` (base) or `pip install typsphinx[pdf]` (includes typst for PDF generation)
- Runtime: Any environment with Python 3.12+

## Entry Points

- Sphinx builder registration: `sphinx.builders` entry points declared in pyproject.toml:
  - `typst = "typsphinx"` - Points to setup() function in `__init__.py`
  - `typstpdf = "typsphinx"` - Same entry point, builder type determined inside

## Package Structure

- Main module: `typsphinx/` (entry point)
  - `__init__.py` - setup() function, config registration
  - `builder.py` - TypstBuilder, TypstPDFBuilder classes
  - `writer.py` - TypstWriter class
  - `translator.py` - TypstTranslator (node visitor)
  - `template_engine.py` - Template loading and rendering
  - `pdf.py` - PDF compilation wrapper
  - `templates/` - Base template (`base.typ`) with Typst Universe imports
