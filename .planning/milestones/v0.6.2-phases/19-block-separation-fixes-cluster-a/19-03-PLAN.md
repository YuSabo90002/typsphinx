<objective>
Close Cluster A with the one fix that does NOT use the shared helper: the `definition_list`
`term`/`definition` separator (FID-05). Set the emitted `terms(...)` call's own `separator:`
parameter to `linebreak()` (D-01: term on its own line, definition indented below) so the term no
longer merges onto its definition when the list is nested in a `list_item` (sub-case a) or the
definition opens with a nested list (sub-case b).

Purpose: FID-05 shares F1's early-return root cause but manifests differently and needs a
structurally different fix (Pitfall 3). Typst's built-in `terms()` separator defaults to a WEAK
`h(0.6em)` horizontal space, not a line break; when the definition's first content is bare inline
(the early-return case) nothing forces a break and it flows onto the term's line. Setting
`separator: linebreak()` unconditionally fixes both sub-cases with a one-line, self-contained
change and — verified — leaves the already-correct `par()`-wrapped case pixel-identical.
Output: FID-05 fix; the def-list gate module extended with two sub-case methods (+ fixtures);
corpus gate confirmed green — completing FID-02..FID-06.
</objective>
