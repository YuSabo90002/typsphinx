<objective>
Establish the shared block-separation idiom for Cluster A and land the two `parbreak()`
(vertical-block) fixes on top of it: consecutive paragraphs inside a list item (FID-02) and
back-to-back body-less confval `desc` nodes (FID-06).

Add the one small shared helper `_emit_forced_break(break_token)` (D-03) — parameterized by
break kind so each of the phase's five sites chooses its own break per the `-b html` authority
(D-01), rejecting both a single uniform break everywhere (D-02) and five fully-independent
zero-sharing fixes (D-04). This plan wires the helper's two `parbreak()` consumers; Plan 02 adds
the two `linebreak()` consumers; Plan 03 handles FID-05 (which does NOT use the helper, per
Pitfall 3).

Purpose: root-cause fix for the dominant v0.6.1-audit finding — in Typst code mode a bare source
`\n`/`\n\n` between two statements is COSMETIC ONLY (zero visual break), so adjacent inline
`text()`/`strong()`/`raw()` results concatenate. The fix emits the real Typst stdlib break the
construct needs.
Output: `_emit_forced_break` helper; FID-02 + FID-06 fixes; two new GATE-01 render-gate modules +
fixtures; full-corpus gate confirmed still green.
</objective>
