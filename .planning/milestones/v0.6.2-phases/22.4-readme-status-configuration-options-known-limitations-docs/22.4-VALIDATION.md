# Phase 22.4 — Validation Strategy

> Per-phase validation contract for feedback sampling during execution.

**Phase shape caveat.** This is a **prose-only** phase — `README.md`, `CLAUDE.md`, the
`pyproject.toml` ruff-ignore comment, and new `.planning/todos/pending/` files. No file under
`typsphinx/` changes, so the GATE-01 real-`typst.compile()` bar that governs every node-handler
phase in this milestone **does not apply here**. The honest verification ceiling is
**content-anchored `grep` assertions** plus the existing suite as a no-regression backstop

every edit shifts the coordinates of every later edit in the same file.

## Wave 0 Requirements

**None.** Existing infrastructure covers all phase requirements.

Per RESEARCH.md Task F, **no new test file is added in this phase** — a deliberate call, not an
oversight. Reasons: (a) the scope fence is prose-only; (b) DOC-01's remedy is *deletion*, so a test
asserting a string's absence has little purchase; (c) the genuinely valuable automation here —
link reachability, which would have caught the discovery #3 404s — needs a
`sphinx-build -b linkcheck` CI job, itself phase-sized work. That linkchecker is filed as a todo
instead. This is the honest ceiling for this phase, and it is recorded rather than papered over.
