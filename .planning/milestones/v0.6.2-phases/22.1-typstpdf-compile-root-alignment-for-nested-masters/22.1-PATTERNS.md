**Core compile pattern to extract into new `compile_typst_file_to_pdf(typ_path, root_dir=None)`** (adapted from lines 129-160, minus the tempfile write):
```python
def compile_typst_file_to_pdf(typ_path: str, root_dir: str | None = None) -> bytes:
    check_typst_available()
    import typst

    try:
        pdf_bytes = typst.compile(typ_path, root=root_dir)
        return pdf_bytes
    except Exception as typst_error:
        error_msg = _parse_typst_error(typst_error)
        logger.error(f"Typst compilation failed at {typ_path}: {error_msg}")
        raise TypstCompilationError(
            message=error_msg, typst_error=typst_error, source_location=typ_path
        ) from typst_error
```

**Error handling pattern** (lines 148-160): identical try/except-wrap-into-`TypstCompilationError` shape reused verbatim — `source_location` becomes the real `typ_path` instead of the temp filename, which is a net error-message improvement noted in D-01's rationale.

**Deprecation-delegate pattern for `compile_typst_to_pdf`** (D-02) — keep the temp-file-write block (lines 133-145) and `finally`-cleanup block (lines 162-168) byte-identical; replace only the inner compile+error-wrap block (129-160) with a call to the new function:
```python
def compile_typst_to_pdf(typst_content: str, root_dir: str | None = None) -> bytes:
    """... .. deprecated:: scheduled for removal in v0.7 ..."""
    check_typst_available()
    temp_file = None
    try:
        temp_dir = root_dir if root_dir else None
        with tempfile.NamedTemporaryFile(
            mode="w", suffix=".typ", dir=temp_dir, delete=False, encoding="utf-8"
        ) as f:
            f.write(typst_content)
            temp_file = f.name
        return compile_typst_file_to_pdf(temp_file, root_dir=root_dir)
    finally:
        if temp_file and os.path.exists(temp_file):
            try:
                os.unlink(temp_file)
            except Exception:
                pass
```
Note: do not use `warnings.warn(DeprecationWarning)` — `pyproject.toml`'s `filterwarnings = ["error::DeprecationWarning", ...]` turns any stray emission into a hard pytest failure across the suite (docstring-only deprecation, per D-02/D-03).

**Core batch/loop pattern to modify** (lines 867-905 — the path resolution prologue at 875-883 is UNCHANGED, this is the Phase 22 invariant that must not drift):
```python
failures: list[tuple[str, str]] = []
for doc_tuple in typst_documents:
    if not doc_tuple:
        logger.warning(f"Skipping malformed typst_documents entry: {doc_tuple!r}")
        continue
    docname = doc_tuple[0]
    stem = self._resolve_output_stem(docname)
    relative_path = self._directory_preserving_relpath(docname, stem)
    typ_file = path.join(self.outdir, relative_path + ".typ")

    if not path.exists(typ_file):
        logger.warning(f"Master document not found: {typ_file}")
        continue

    try:
        # typ_file IS the real on-disk path -- no read-to-string, no temp copy
        pdf_bytes = compile_typst_file_to_pdf(typ_file, root_dir=self.outdir)
        pdf_file = path.join(self.outdir, relative_path + ".pdf")
        with open(pdf_file, "wb") as f:
            f.write(pdf_bytes)
        logger.info(f"Generated PDF: {pdf_file}")
    except Exception as e:
        logger.error(f"Failed to compile {typ_file}: {e}")
        failures.append((docname, str(e)))

if failures:
    summary = "; ".join(f"{docname}: {err}" for docname, err in failures)
    raise ExtensionError(
        f"typstpdf: {len(failures)} master document(s) failed to compile "
        f"to PDF: {summary}"
    )
```

**Error handling pattern change (D-04/D-05):** current bare `except Exception: logger.error(...)` swallow (lines 904-905) becomes per-master append-to-`failures` + aggregate raise after the loop. New import needed: `from sphinx.errors import ExtensionError` (per RESEARCH.md recommendation; not currently imported anywhere in the codebase — verify no name collision).

**Path-resolution invariant to preserve exactly:** `write_doc()` (`builder.py:808-810`) and `finish()` (`builder.py:881-883`) must keep computing `stem`/`relative_path` via the identical `_resolve_output_stem` + `_directory_preserving_relpath` expression — do not re-derive independently.
