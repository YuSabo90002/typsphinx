<objective>
Restore object-description field spacing for FID-09 with two small, independent edits in
`typsphinx/translator.py`: (a) `depart_field_name`'s `text(":")` becomes `text(": ")`
(colon-space, D-01's central edit), and (b) `depart_field` (currently `pass`) emits a standalone
two-space statement between sibling fields via the established `node.next_node(descend=False,
siblings=True)` idiom — targeting the exact ROADMAP SC#3 string `"Type: int (a number)  Default:
42"` (a deliberate DOUBLE space between fields).

Purpose: `:type:`/`:default:` fields currently render "Type:int (a number)Default:42" — the
colon has no trailing space and sibling fields merge with no boundary. The colon-space fix is a
one-literal change; the inter-field boundary reuses the same "am I the last sibling" idiom
`depart_desc_parameter` already uses for comma-separation.
Output: The two source edits, a new real-compile GATE-01 fixture (structural + `%PDF` + pypdf
adjacency, D-07), and the required update to the ONE pre-existing test that locks the old
no-space colon form.
</objective>
