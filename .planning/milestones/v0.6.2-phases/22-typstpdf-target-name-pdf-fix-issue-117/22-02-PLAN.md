Create `tests/test_target_name_render_gate.py` as a real-compile GATE-01 module, structurally
mirroring `tests/test_cross_doc_label_namespace_render_gate.py`.

Module docstring: state that this is the fast offline regression gate for Issue #117 / PDF-01 —
`typst_documents`' target name (tuple element `[1]`) governs the emitted `.typ` AND `.pdf`
filenames, not the source docname. State that it drives `-b typstpdf` rather than `-b typst` on
purpose, because the `.pdf` half of the contract only materializes inside
`TypstPDFBuilder.finish()`'s `typst.compile()` call. State the two-directional design explicitly:
the target-named artifacts must be PRESENT and the docname-named artifacts must be ABSENT, so the
gate fails on both counts against the pre-fix builder (which emitted only the docname-named pair).

Imports and probes, copied from the analog: `subprocess`, `sys`, `pathlib.Path`, `pytest`, and the
`try: import typst  # noqa: F401` / `TYPST_AVAILABLE` boolean probe.

Fixtures: `target_name_render_gate_source_dir` returning
`Path(__file__).parent / "roots" / "test-basic"` — note the `roots` segment, NOT `fixtures`, because
`test-basic` lives directly under `tests/roots/`. And `temp_build_dir` returning `tmp_path / "_build"`,
identical to the analog.

Build driver: a module-private `_run_sphinx_build_typstpdf(source_dir, build_dir)` returning
`subprocess.run([sys.executable, "-m", "sphinx", "-b", "typstpdf", str(source_dir), str(build_dir)],
capture_output=True, text=True)`. Carry over the analog's docstring explaining that
`sys.executable -m sphinx` is used deliberately instead of a resolved `sphinx-build` binary, to
reuse the exact interpreter/venv running the test and sidestep the NixOS-sandbox PATH-shadowing
hazard.

Test class `TestTargetNameRenderGate`, decorated with
`@pytest.mark.skipif(not TYPST_AVAILABLE, reason=...)`, containing one test method
`test_typstpdf_emits_target_named_artifacts_and_not_docname_named(self,
target_name_render_gate_source_dir, temp_build_dir)`.

Test body, organized as four banner-commented concerns following the analog's `# (n) LABEL:` style:

`# (1) BUILD:` run the driver, assert `result.returncode == 0` with `stdout`/`stderr` interpolated
into the failure message. Then guard against the failure mode where `finish()` logs rather than
raises: assert the substring `Typst compilation failed` is not in `result.stderr`, and assert the
substring `Master document not found` is not in `result.stderr` — the latter is the precise signal
that `finish()`'s read-back path and `write_doc`'s write path disagreed, which is the exact
regression this phase's shared-helper factoring exists to prevent.

`# (2) TARGET-NAMED .typ PRESENT:` assert `(temp_build_dir / "output.typ").exists()` with a failure
message naming `tests/roots/test-basic/conf.py`'s target entry, and assert its `stat().st_size > 0`.

`# (3) TARGET-NAMED .pdf PRESENT AND VALID:` assert `(temp_build_dir / "output.pdf").exists()`,
assert `stat().st_size > 0`, and open it in binary mode and assert the first four bytes equal the
PDF magic byte string, exactly as the analog does at its lines 222-230.

`# (4) DOCNAME-NAMED ARTIFACTS ABSENT:` assert `not (temp_build_dir / "index.typ").exists()` and
`not (temp_build_dir / "index.pdf").exists()`, each with a failure message explaining that this is
the D-08 clean-break requirement — the builder must stop emitting the docname-named pair when a
target name differs, not emit both. Include in the message that a failure here against a passing
concern (2)/(3) means a compatibility shim was introduced.

Do not assert on the wording of any `typst.compile()` diagnostic; assert on the process return code
and on emitted artifacts only. Do not modify `tests/roots/test-basic/` in any way.

Keep Python 3.10 compatibility, line length 88 (black).
