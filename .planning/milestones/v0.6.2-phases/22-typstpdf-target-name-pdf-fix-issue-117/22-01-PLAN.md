<objective>
Give `typst_documents`' target name (tuple element `[1]`) its first consumer in the package: a single
shared normalization helper on `TypstBuilder`, then wire all three output-path sites to it so the
emitted `.typ` and the compiled `.pdf` always agree.

Purpose: `doc_tuple[1]` is currently read nowhere in typsphinx. Both write sites and the PDF
read-back site derive filenames from the docname, so `typst_documents = [('index', 'manual.typ', ...)]`
emits `index.typ` / `index.pdf` — Issue #117, and a violation of the contract published at
`docs/configuration.rst:43`. Per D-01 this plan fixes BOTH filenames, not the PDF alone.

Output: `TypstBuilder._resolve_output_stem()` plus three rewired call sites in `typsphinx/builder.py`,
and `tests/test_builder_output_stem.py` covering every locked normalization rule.
</objective>
