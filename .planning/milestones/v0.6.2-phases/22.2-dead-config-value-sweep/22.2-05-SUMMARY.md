- The GATE-01 fixture's `typst_template_mapping` is deliberately a single-key mapping (`{"project": "title"}`), not the full default mapping, so the fixture simultaneously proves "nothing else is back-filled" (D-05) and "explicit wins on collision" (D-08/BUG-E) with one configuration.
- The config->output difference matrix uses `-b typst` (source-text comparison only, faster); the primary per-defect gate and the D-12 example gate use `-b typstpdf` (a real compile) -- real-compile coverage is not duplicated where it adds no discriminating power.
- `approach2`'s D-12 test asserts package-actually-used in BOTH directions (positive: pinned import present; negative: default-template marker absent), per the owner ruling in `22.2-06-PLAN.md` that a build-exit-0 assertion alone is insufficient.

## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 1 - Bug] Class-scoped pytest fixtures declared as instance methods**
- **Found during:** Task 1 (first test run of `TestPackageOnlyConfigGate`)
- **Issue:** This project's pytest 9.1.1 + `error::DeprecationWarning` filter (`pyproject.toml`) escalates `PytestRemovedIn10Warning` (a `DeprecationWarning` subclass) to a hard error. A class-scoped fixture defined as a plain instance method (`def build(self, tmp_path_factory)`) triggers this warning because each test method gets its own class instance while the fixture itself only runs once per class -- state written on `self` would not be visible across tests. All three class-scoped build fixtures in this plan's two new modules were affected.
- **Fix:** Declared `build`, `emitted_master_text`, and `baseline_text` as `@staticmethod` (in addition to `@pytest.fixture(scope="class")`), since none of them mutate `self` -- they return a plain dict/string value, so the staticmethod form is correct, not merely warning-suppression.
- **Files modified:** `tests/test_package_only_config_gate.py`
- **Verification:** All 12 tests in the module pass under the project's strict warning filter.
- **Committed in:** `4ae868f`, `46b1343` (part of the respective task commits)
