## FLAGGED PLANNER ASSUMPTION — route to backlog, do NOT fix here

Verified during planning: the translator has **no `visit_citation` / `visit_label` handler**. For a
reStructuredText citation it emits two adjacent expressions with no separator inside a Typst code
block, and the resulting document fails to compile with a syntax error. An in-code comment at
`typsphinx/translator.py:2257` records that citation nodes were deliberately left untouched by an
earlier phase, so this is a known, unimplemented feature rather than a regression — but it is
tracked nowhere in `.planning/`. This plan works around it by removing the citation constructs from
the bundled sample; implementing citation support is a translator feature squarely outside a
config-sweep phase. **Record it in the SUMMARY for backlog routing.**
