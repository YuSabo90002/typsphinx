## Area selection

All four proposed gray areas were selected. `typst_output_dir` removal (CONF-01) was **not**
presented — it was already decided by the owner on 2026-07-21 (no deprecation period) and carried
forward unchanged.

Before questioning began, a defect not present in the roadmap catalogue was surfaced: `render()`'s
four `@preview` imports and `codly-init` live only inside the `template_file` branch
(`template_engine.py:310-317`), so a naive BUG-A fix would leave the package-only path with no
`codly` / `mitex` / `gentle-clues` at all. Recorded as BUG-F.

## A. Package-only output shape (BUG-A)

### A1 — BUG-A fix shape

| Option | Description | Selected |
|--------|-------------|----------|
| writer-side branch | `typst_package` set + `typst_template` unset → `render(template_file=None)`; builder's early return stays | ✓ |
| builder writes a re-export shim `_template.typ` | Unifies the import path but leaves a contentless intermediate file in outdir and keeps the 22.1 depth-computation dependency | |
| Demote package-alone to unsupported | Contradicts CONF-02's success criterion; would need a roadmap amendment | |

**User's choice:** writer-side branch (recommended)

### A2 — Essential `@preview` imports on the package-only path

| Option | Description | Selected |
|--------|-------------|----------|
| Always emit | Master output always leads with the 4 imports + `codly-init` + `codly(languages:)`, matching the included-document path at `writer.py:139-150` | ✓ |
| Emit only on the package-only branch | Zero byte-diff on existing output, but a 4th copy of the import list and a wider `@preview` version-sync surface | |
| Do not emit — users declare via `typst_package_imports` | Smallest generated output, but any code block breaks on undefined symbols | |

**User's choice:** Always emit (recommended)

### A3 — Detecting `typst_package` + `typst_template`

**First formulation was withdrawn.** The original question assumed the combination was a supported
"wrapping" route and that `advanced.rst:126-133` misattributed its failure. The user challenged the
premise ("仕様上兼用できなくないか？"). Verification confirmed the user was right: commit `1ae8c52`
already replaced that note with an accurate one stating *use `typst_package` **or** `typst_template`
— not both*, and naming the real mechanism. **ROADMAP SC#3's description of that note is stale**;
SC#3's remaining substance is the `advanced.rst:59` modern-cv example only. The question was
reformulated to ask about code-level enforcement instead.

| Option | Description | Selected |
|--------|-------------|----------|
| Warn + `typst_template` wins | Narrow the builder early-return to "`typst_package` set AND `typst_template` unset" so the wrapper is written; matches documented intent | ✓ |
| Warn + `typst_package` wins | Faithful to the current branch, but silently discards a template the user wrote | |
| No detection, docs only | Minimal diff, but keeps the one silently-broken configuration alive | |

**User's choice:** Warn + `typst_template` wins (recommended)

## C. Author configs (BUG-C) and parameter precedence (BUG-E)

**Two withdrawn formulations.** The first ask about the two author configs was interrupted by the
user asking which one the examples actually use ("例は authors の方か？"). Measurement answered it:
`typst_authors` appears in every user-facing doc and in both bundled examples; `typst_author_params`
appears in **zero** user-facing docs or examples — only in `CLAUDE.md:67`'s inventory and in code,
with one backward-compat unit test. Since neither has ever affected output, the "backward
compatibility" never existed.

The second formulation was interrupted again, by the user asking what happens when
`typst_template_function` and the author configs are both defined, and stating that
`typst_template_function` ought to win. Verification confirmed a real defect: `render()`
(`template_engine.py:336-338`) applies `typst_template_params` first and auto-derived params second,
so explicit user config is silently clobbered. Recorded as **BUG-E** — not in the roadmap catalogue.

### C1 — BUG-E precedence

| Option | Description | Selected |
|--------|-------------|----------|
| Invert precedence | Auto-derived first, `typst_template_function["params"]` last, so explicit config wins | ✓ |
| Invert + warn on key collision | Visible, but noisy for intentional overrides | |
| Record only, defer to backlog | Scope-safe, but leaves the template path silently clobbering | |

**User's choice:** Invert precedence (recommended)

### C2 — Fate of the two author configs

| Option | Description | Selected |
|--------|-------------|----------|
| Wire `typst_authors`, delete legacy | `typst_authors` becomes live and real-compile-verified; `typst_author_params` removed entirely and folded into CONF-01 | ✓ |
| Wire `typst_authors`, leave legacy registered | Smaller diff, but leaves one instance of exactly the property this phase exists to eliminate | |
| Delete both | Smallest code surface, but `charged-ieee`'s `ieee()` takes author dicts, so the bundled example loses most of its value | |

**User's choice:** Wire `typst_authors`, delete legacy (recommended)
