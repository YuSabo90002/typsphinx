# Phase 22.2 Plan 06: Repair the approach2 bundled sample Summary

**Relocated approach2's custom Typst template wrapper under `source/` so its configured path actually resolves, fixed two compile-blocking Typst defects inside it (an invalid `.lower()` method call and a bibliography argument naming a nonexistent file), and excised the untranslatable RST citation constructs from its document — turning a silently-fallback-to-default, warning-then-continue sample into a warning-free real-compile demonstration of the D-03-promoted custom-template-plus-package route.**

## Accomplishments
- `examples/charged-ieee/approach2/_templates/_template.typ` relocated to `examples/charged-ieee/approach2/source/_templates/_template.typ` via `git mv`, restoring the source-directory-relative resolution invariant shared by `writer.py`/`builder.py`/the published docs/the other two bundled examples — with zero edit to `conf.py`.
- Fixed the wrapper's author-email construction: `name.split(" ").at(0).lower()` (a nonexistent Typst string method) → `lower(name.split(" ").at(0))` (the correct free-function form).
- Removed the `bibliography: "refs.bib"` argument from the `ieee.with()` show-rule (the file exists nowhere in the repo and nothing produces one at build time), replacing it with a comment recording why.
- Removed the RST citation directive block, its "References" heading, and the inline `[Krizhevsky2012]_` reference from `index.rst`, rewording the CNN paragraph so it still reads naturally, and adding a top-of-file RST comment (worded consistently with the sibling excision in plan 22.2-02) explaining the translator has no citation-node handler.
- Verified end-to-end with a real `sphinx-build -b typstpdf` compile: `build succeeded.`, **zero** warnings, a 68 778-byte `%PDF-1.7` PDF produced, and the emitted `_template.typ` provably the sample's own wrapper (3 occurrences of the pinned package import, 0 occurrences of the bundled default template's marker text).
