<objective>
Repair the `TemplateEngine` parameter pipeline: emit the essential `@preview` imports for every
master (D-02), stop default parameter injection on the package path (D-05), wire `typst_authors`
into the emitted parameters as a native structure (D-07), invert the parameter precedence so
explicit configuration wins (D-08), and delete the legacy author-params surface end-to-end (D-07,
CONF-01).

Purpose: CONF-02's engine half. Four of the six confirmed defects (BUG-B, BUG-C, BUG-E, BUG-F) all
live in `map_parameters()` and `render()`, which are the single assembly point for what a master
document's `.typ` actually says. They are grouped into one plan because they touch the same two
functions and cannot be parallelised without collision.

Output: a template engine whose emitted parameters are exactly what the user asked for on the
package path, whose author configuration is live, and whose explicit configuration is authoritative.
</objective>

## LOAD-BEARING CORRECTION — read before Task 1

`22.2-RESEARCH.md` Common Pitfall #1 correctly identifies that the apply-mapping loop (driven by
`self.parameter_mapping` defaulting to the class-level default mapping) is an injection source that
CONTEXT.md's D-05 citation missed. **But its conclusion that the fill-if-missing back-fill then
"becomes correctly inert" is WRONG.** Verified by execution against the current tree during
planning: setting `parameter_mapping` to an empty dict still yields
`{'title': '', 'authors': (), 'date': None}`, because the back-fill block runs unconditionally and
does not consult the mapping at all. A `date` argument would therefore still be emitted and the
package function would still reject it.

**D-05 requires BOTH gates.** Task 1 must gate the constructor-level mapping default AND the
fill-if-missing back-fill. Gating either one alone leaves BUG-B alive.

| developer `conf.py` → typsphinx | Already fully trusted arbitrary Python; unchanged by this plan. |

| T-22.2-07 | Tampering (content injection into generated Typst source) | `map_parameters()` author wiring, `render()` parameter loop | medium | mitigate | The new author wiring places NATIVE Python values into the parameter dict so they flow through the existing `_format_typst_value()` escaping (which escapes backslashes and double quotes on every string) rather than bypassing it.
