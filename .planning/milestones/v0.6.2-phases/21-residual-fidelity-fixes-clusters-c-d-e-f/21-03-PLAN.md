  external ref emits `link("url", …)` in `index.typ`.

- **FID-13 boundary half (D-03):** a stray double space appears before a following period/word after a
  named external reference. Root cause (RESEARCH.md Pattern 5 / Pitfall 2, verified — corrects CONTEXT.md's
  `visit_reference` pointer): `visit_target`'s `_in_reference_with_target` branch prepends a `\n` before an
  invisible `#label(...)` call, and a newline in Typst MARKUP mode renders as a visible space that combines
  with the genuine following space → double space. Fix: drop the leading `\n` (the preceding content is
  always the closing `)` of the `link(...)` call, so `)#label(` needs no separator). Verified Shape-B
  (pypdf, D-09): single space present, double space absent.
