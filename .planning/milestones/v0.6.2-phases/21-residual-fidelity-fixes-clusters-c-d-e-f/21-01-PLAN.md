<objective>
Fix the two "wrapping / markup-vs-code-mode" residual fidelity findings, each isolated to its own
`typsphinx/translator.py` handler, and ship a real-`typst.compile()` GATE-01 fixture per finding:

- **FID-10 (Cluster C — margin overflow):** a long run of colon-leading inline `literal` role tokens
  (`:cpp:any:` `:cpp:class:` …) overflows the right margin and clips mid-token = content loss. Root
  cause (RESEARCH.md, verified against a real compile): Typst's UAX14 LB13 suppresses a line break
  *before* a colon even when a real breakable `text(" ")` token precedes it. Fix per D-04 (boundary-only,
  never break inside a token): insert a leading zero-width space (U+200B) inside the `raw()` content in
  the NON-`in_table` branch of `visit_literal`, gated to a narrow no-break-before character class so the
  44 existing exact-match `raw("...")` assertions stay byte-unchanged (Assumption A1 — this plan RESOLVES
  it with an explicit real-compile verification step, per the D-06 escape-hatch spirit). The existing
  `self.in_table` ZWSP primitive stays isolated (D-05).

- **FID-12 (Cluster E — codly config leak):** a `literal_block` with BOTH a `:caption:` AND nested in a
  `list_item` leaks its codly config wrapper as visible text. Root cause (RESEARCH.md D-Disc-2, verified
  against a real compile): the list-item `{ }` wrapper's own OPENING brace is unconditionally bare, but a
  bare `{` inside the captioned figure's markup-mode content is parsed as literal text — it needs `#{` to
  re-enter code mode. Fix: make the wrapper's opening brace carry `#` iff captioned AND in a list item;
  the existing `codly_prefix` predicate is already correct and stays unchanged.

Purpose: the tail of the coherent v0.6.2 translator-fix series — small, independent, low-blast-radius edits.
Output: two `translator.py` edits + two new render-gate fixtures + two new Shape-B (structural + pypdf)
gate modules. Verification per D-07/D-09. Milestone invariant honored throughout.
</objective>
