## Test Infrastructure

| Property | Value |
|----------|-------|
| **Framework** | pytest (config in `pyproject.toml`, `[tool.pytest.ini_options]`) |
| **Config file** | `pyproject.toml` — `testpaths = ["tests"]`, markers `slow`/`integration`, `filterwarnings = ["error::DeprecationWarning", "error::PendingDeprecationWarning", …]` |
| **Quick run command** | `uv run pytest tests/test_pdf_generation.py tests/test_nested_master_render_gate.py tests/test_builder_output_stem.py tests/test_target_name_render_gate.py -x` |
| **Full suite command** | `uv run pytest` |
| **Estimated runtime** | ~20s quick · ~4–8 min full (GATE-01 fixtures perform real `typst` compiles and may fetch `@preview` packages) |

> **Worktree note (CLAUDE.md § Worktree-isolated execution):** `workflow.use_worktrees` is `true`
> and worktree isolation is the standing execution mode. Provision the worktree first with
> `env -u VIRTUAL_ENV -u UV_PROJECT_ENVIRONMENT uv sync --extra dev`, then run **every** command
> above via `uv run`. Without this, pytest imports the unchanged main-tree package and gates stay
> red after a correct fix.

## Sampling Rate

- **After every task commit:** `uv run pytest tests/test_pdf_generation.py tests/test_nested_master_render_gate.py tests/test_builder_output_stem.py tests/test_target_name_render_gate.py -x` — fast, no network, covers every file this phase touches or must not break
- **After every plan wave:** `uv run pytest -m "not slow" -x` **plus** the new GATE-01 module explicitly (`uv run pytest tests/test_<new>_gate.py -v`)
- **Before `/gsd-verify-work`:** Full suite green (`uv run pytest`), including `tests/test_corpus_gate.py` and `tests/test_preview_version_sync.py`, which must stay **untouched and green** per the milestone invariant (SC#5 — no `@preview` bump, 3-way version-sync surface unchanged)
- **Max feedback latency:** ~20 seconds (quick command)
