# Phase 22.3 Plan 02: De-couple Nested-Master Render Gate from typst-py Wording Summary

**All six `typst-py` diagnostic-string literal assertions removed from `tests/test_nested_master_render_gate.py`, replaced with structural preconditions plus a real three-part ablation that proves the include/image failure class by compiling a fully-neutralized copy to actual PDF bytes.**

## Performance

- **Duration:** ~20 min
- **Completed:** 2026-07-22T13:00:52Z
- **Tasks:** 3
- **Files modified:** 1

## Accomplishments

- `test_outdir_root_compile_basis_still_fails` (the template-reference RED-only differential) no longer reads `typst-py`'s error text at all: the removed `"escape"` and `"_template.typ"` membership checks are replaced by the pre-existing emitted-source precondition plus a new byte-identity assertion (`basis_copy_bytes == emitted_master_bytes`), which pins the test's actual premise -- nothing was rewritten, so only the compiled file's directory depth varies.
- `test_sibling_include_fails_at_outdir_root_and_resolves_in_place`'s RED half (Step 4) no longer reads the compiler's message either: the four removed literal checks (`"usage.typ"`, `"not found"`, negative `"_template.typ"`, negative `api_usage_segment`) are replaced by a filesystem precondition -- `usage.typ` asserted absent at the outdir root before the compile -- which, combined with the Step 2/3 emitted-source preconditions, makes a sibling-include failure the only explicable outcome.
- A new Step 6 adds the full three-part ablation the plan requires (D-08/D-09/Pitfall 3): rewrite the single `image("../logo.png")` occurrence to `image("logo.png")` (pinned by two stdlib-only AST probes matching the literal `.count(...)`/`.replace(...)` argument nodes), copy `usage.typ` to the outdir root (`logo.png` is already there from the normal `-b typst` asset-copy step, so it is not re-copied), write the fully-neutralized text as `_prefix_basis_fully_neutralized.typ`, and compile it -- asserting the result starts with `b"%PDF"`. This is the positive, discriminating half of the D-08 proof: a RED-only result would not have satisfied the plan.
- The captured RED-half exception (`red_half_error`) is consumed only inside the ablation's `startswith(b"%PDF")` failure message, never inside an asserted expression -- preserving the diagnostic for humans without re-coupling the test to it (WR-02 / D-10).
- Both module-level docstring paragraphs describing the removed literal assertions were rewritten to describe the new structural-precondition / ablation design; the historical Typst error text that remains in the module docstring is explicitly labelled as a non-contract record (an upstream rewording is expected NOT to break either test).
- Full module green (4/4 tests); zero new `typsphinx/` diff; `black`/`ruff` clean; non-slow suite matches the pre-change baseline exactly modulo the known worktree-only environmental exclusion set (see Issues Encountered).

## Decisions Made

- Kept two test functions rather than merging them (discretion_record, D-09 second half): `test_outdir_root_compile_basis_still_fails` stays the RED-only differential (its subject fixed by elimination against the sibling test's ablation), and `test_sibling_include_fails_at_outdir_root_and_resolves_in_place` carries the full GREEN ablation as its Step 6, in addition to its pre-existing Step 5 compile-in-place proof.
- Each captured diagnostic string (`str(exc_info.value)` in test 1, `red_half_error` in test 2) is attached to a meaningful, text-free assertion's failure message (the byte-identity check; the ablation's `%PDF` check) rather than left as a dead local -- this project's ruff config does not flag `F841` (unused locals), so this was a deliberate choice per the plan's discretion_record, not something a linter would have caught.
- Line 248's pre-existing `assert "file not found" not in result.stderr` (in `test_typstpdf_nested_master_resolves_include_and_image`) was left untouched, per the plan's `flagged_assumptions`: it is a negative assertion on stderr whose primary pass/fail signal is `returncode == 0`, so an upstream rewording would make it silently vacuous rather than turn CI red -- it is not one of D-10's six measured literals and is out of this plan's authorized scope.
