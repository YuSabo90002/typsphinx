# Phase 22.3 Plan 03: GATE-01 Must-Fail Subprocess Gate for WR-01 Summary

**A new `sphinx-build -b typstpdf` subprocess gate (`tests/test_missing_and_malformed_master_gate.py`) proves, over a mixed-bad-entries fixture, that a build now exits non-zero for an unknown-docname master and a malformed `typst_documents` entry while the fixture's valid master still gets its PDF — the suite's first must-fail GATE-01 module, and the subprocess-level half of ROADMAP SC#5's pre-fix must-fail proof.**

## Accomplishments

- Created `tests/fixtures/missing_and_malformed_master_gate/` — a minimal Sphinx project whose `typst_documents` holds, in load-bearing order, one valid master (`index`), one docname absent from `self.env.found_docs` (`ghost`, with no `ghost.rst` and no toctree reference), and one malformed empty tuple (`()`). No `typst_template`/`typst_package` is set, keeping this a pure WR-01 gate that compiles through the bundled default template.
- Created `tests/test_missing_and_malformed_master_gate.py` — the suite's first must-fail subprocess gate. Two tests: one proves the mixed build exits non-zero while `index.typ`/`index.pdf` still exist and no `ghost.typ`/`ghost.pdf` exists; the other proves the aggregate message's `ghost` fragment uses D-04's `found_docs`-discriminating text (never the `Master document not found` fallback), carries D-07's `(): malformed typst_documents entry` identifier, and reports both failures in `typst_documents` iteration order (`ghost` before `()`).
- Recorded the subprocess-level half of ROADMAP SC#5's pre-fix must-fail proof: temporarily reverted the two `failures.append(...)` call sites plan `22.3-01` added, ran the new gate (both tests failed, observed `returncode == 0`), then restored `typsphinx/builder.py` byte-for-byte from a scratch backup and re-ran the gate (both tests passed). `git diff --stat typsphinx/builder.py` produced no output afterward.
- Ran the full non-slow suite (worktree-local count) and the corpus/preview-sync gates; both green with no fixture regenerated and zero net changes to `typsphinx/`.

Per this project's CLAUDE.md critical-environment-rules guidance and the `nixos-sandbox-test-env` memory note, this is flagged for the orchestrator's post-merge gate (which runs on the main tree, where the hazard does not apply) rather than treated as a regression introduced by this plan.
