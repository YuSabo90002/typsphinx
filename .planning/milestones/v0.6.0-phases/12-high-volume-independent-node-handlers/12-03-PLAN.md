<objective>
Land DESC-01…04: the four autodoc signature sub-part handlers, all small additive extensions to the
existing `desc_*` visitor family in `typsphinx/translator.py`, plus a real-compile GATE-01 fixture.
DESC-01 (`desc_returns`) injects a literal ` -> ` arrow; DESC-03 (`desc_optional`) literal-bracket-wraps
optional parameters, recursion-safe via the existing `_desc_parameter_has_content` flag; DESC-04
(`desc_inline`) is a transparent pass-through that resolves D-06 by node-type dispatch (no strong()
wrapper); DESC-02 (`desc_signature_line`) inserts a real Typst `linebreak()` between signature lines
using one new `_is_first_desc_signature_line` state variable — the phase's single D-05-sanctioned new
state var.

Purpose: these complete autodoc signature rendering (return arrows, multi-line breaks, nested optional
brackets, inline fragments). The critical, empirically-proven subtlety is DESC-02: a source-level `\n`
between code-mode statements produces ZERO visual line break (confirmed via `typst.compile()` + `pypdf`);
only Typst's `linebreak()` (stdlib, first use in this codebase, no new import) produces a real break.
DESC-02 also carries the phase's one open question (Assumption A1): the exact rST syntax producing a
genuine multi-`desc_signature_line` doctree is unconfirmed — so this plan makes "dump a real multi-line
signature doctree" an explicit first implementation step.

Output: `visit_desc_returns`/`depart_desc_returns`, `visit_desc_optional`/`depart_desc_optional`,
`visit_desc_inline`/`depart_desc_inline`, `visit_desc_signature_line`/`depart_desc_signature_line`,
the `_is_first_desc_signature_line` state var + `visit_desc_signature` reset, one fixture project, and
one real-compile test class.

**Requirements coverage:** DESC-01 (D-05), DESC-02 (D-05), DESC-03 (D-05), DESC-04 (D-06). GATE-01
standing bar extended (SC#5).
