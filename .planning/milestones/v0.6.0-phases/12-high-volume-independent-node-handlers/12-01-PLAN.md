<objective>
Land VER-01: render the four version directives (`versionadded`/`versionchanged`/`deprecated`/`versionremoved`)
as an unboxed italic label + inline body, matching Sphinx's own `versionlabels` wording, and ship a
real-compile GATE-01 fixture proving it. Two additions to `typsphinx/translator.py`: a transparent
`visit_versionmodified`/`depart_versionmodified` pass-through pair (silences the ×972 unknown_visit
warning), and a one-branch extension of the existing `visit_inline`/`depart_inline` that detects the
`versionmodified` CSS class and delegates to the file's proven dummy-node emphasis idiom.

Purpose: `versionmodified` is the single most-frequent dropped node in Sphinx's own docs (×972).
The label text is already computed by Sphinx's `VersionChange` directive at parse time and baked into
the doctree as `nodes.inline(classes=["versionmodified", <kind>])` — so the translator only needs to
italicize the already-present classed inline, not import or reconstruct anything (research superseded
D-01's speculated `sphinx.locale`/`versionlabels` import — document that in the code comment).

Output: `visit_versionmodified`, `depart_versionmodified`, the `visit_inline`/`depart_inline`
classed-dispatch branch, one fixture project, and one real-compile test class.

**Requirements coverage:** VER-01 (per D-01 intent + D-02 layout). GATE-01 standing bar extended (SC#5).

**Wave note:** `depends_on: []` — this is wave 1. Waves 2-4 (Plans 02-04) are sequenced after this one
purely because they share `typsphinx/translator.py` and `tests/test_pdf_render_gate.py` (file-ownership),
NOT because of any logical dependency — the four handler groups are genuinely independent.
</objective>
