<objective>
Generalize the load-bearing `visit_title`/`depart_title` dispatch in `typsphinx/translator.py` so a
`.. topic::` renders as a titled `clue` aside (D-01), a `.. contents::` renders box-less as a bold
label above its resolved bullet list (D-05), and every title is safe against the two currently-live
`visit_title` fatals — the level-0 heading class (D-06) and the pre-existing multi-child-title
concatenation fatal (RESEARCH.md Pitfall 1, an approved in-scope addition). Add `visit_topic`/
`depart_topic` (D-01/D-02) so the widened buffer-swap branch's captured title is actually consumed.

Purpose: BLK-02 — a `.. topic::` becomes a titled aside reusing the admonition helper, with heading/
TOC structure unchanged. This is the "shared dispatch-point generalization" the whole phase is named
for; because `visit_title` is depended on by every admonition and section heading, the change must be
minimal, additive, and non-regressive.

Output: generalized `visit_title`/`depart_title`, new `visit_topic`/`depart_topic`, one new instance
flag (`_topic_is_contents`), and dedicated unit tests in `tests/test_topics.py`.

**Atomicity (Pitfall 2 + mandatory scope rule):** D-01, D-02, D-05, D-06 and the Pitfall-1 fix ALL
edit `visit_title`/`depart_title` (or the methods that consume its buffer) and MUST land together in
this one plan/wave. Splitting D-02 from D-01 would capture a title buffer that nothing consumes (a
worse regression than baseline). The GATE-01 real-compile proof for this plan lives in the Wave-3
combined render gate (13-03); this plan's own bar is the unit tests plus a green existing admonition
suite (regression guard).
</objective>
