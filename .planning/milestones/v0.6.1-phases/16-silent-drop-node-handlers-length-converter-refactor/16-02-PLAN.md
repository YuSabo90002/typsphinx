<objective>
Implement `visit_manpage`/`depart_manpage` in `typsphinx/translator.py` so the `:manpage:` role
renders its literal page-reference text (e.g. `ls(1)`) in italic, Sphinx-HTML-faithful (per D-02),
by delegating to the existing `visit_emphasis`/`depart_emphasis` pair вАФ and prove it through a new
GATE-01 real-compile fixture exercising a plain paragraph, a list item, and a figure caption.

Per D-02a: NO linkification вАФ `manpages_url` is a separate Sphinx feature, out of scope; the
handler renders literal text only (the fixture asserts no link() is emitted).

Implementation nuance the executor must respect (16-RESEARCH.md, verified by source read): CONTEXT
D-02's literal `#emph[`вА¶`]` phrasing understates the real codebase pattern. The established italic
wrapper is `visit_emphasis`'s `{prefix}emph({`вА¶`})` вАФ a code-mode call with a conditional `#`
prefix and a ~50-line separator/concat-context state machine. Delegation reuses all of it for free
and is the locked recommendation; a bespoke pair is NOT permitted unless it replicates every piece
of that state handling (16-RESEARCH.md Pitfall 4 / Anti-Patterns).

Purpose: MAN-01 вАФ eliminate the `unknown node type: <manpage>` silent drop (√Ч10 in the corpus).
Output: two new translator methods, a new fixture project, a new slow-marked render-gate test class.
</objective>
