<objective>
Add Typst-native footnote support to `typsphinx/translator.py` (requirement FN-01): a
document-order pre-pass in `visit_document` indexes footnote bodies by id (D-01), `visit_footnote`
suppresses the definition node at its natural location (D-05), and `visit_footnote_reference`
emits the note inline at the reference site — the bracket-wrapped definition form on first citation
and the bare reuse form on repeat citation (D-02/D-03/D-04/D-06), with a warn+skip guard for
dangling references (D-08). `citation`/`citation_reference` remain untouched (D-07); a
defined-but-never-referenced footnote is silently dropped (D-09). Ships with a dedicated
string-assertion unit module (`tests/test_footnotes.py`) written test-first.

Purpose: This is the only architecturally-new item of the milestone — the document-order pre-pass.
The exact compile-safe emission syntax is already PROVEN in 14-RESEARCH.md via 17 real
`typst.compile()` runs; this plan lands that proven design, do not re-derive it.
Output: three new/edited translator handlers + one new unit-test module. The real-compile GATE-01
proof is Plan 14-02 (Wave 2).
</objective>
