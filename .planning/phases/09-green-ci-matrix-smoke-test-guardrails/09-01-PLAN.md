<objective>
Add a `typst compile` smoke test (CI-02, D-04/D-05) that exercises ALL FOUR bundled `@preview` packages
(`codly`, `codly-languages`, `mitex`, `gentle-clues`) in one minimal Sphinx fixture, compiles the emitted
`.typ` with `typst.compile()`, and lets any `typst.TypstError` (e.g. `unknown variable: kai`) fail the test
loudly. This closes the coverage gap the internal-only `test_preview_version_sync.py` misses and that the
existing `test_pdf_render_gate.py` (admonition/gentle-clues-only fixture) never covers for mitex.

Purpose: The `kai` break was a mitex runtime error. mitex's `mi()`/`mitex()` functions are only invoked when
a `.. math::` directive or `:math:` role is present — the existing gate's fixture has neither, so it would
NOT catch a mitex regression. A fixture with real math content is the real guard before release.

Output: A new fixture (`tests/fixtures/preview_smoke/`) + a new pytest module
(`tests/test_preview_smoke_gate.py`) that rides the existing 3-OS × Python 3.12–3.13 matrix for free, plus a
documented one-time negative-control proof of the fixture's sensitivity.
</objective>
