<objective>
Fix the admonition markup/code-mode mismatch at its single choke point in
`typsphinx/translator.py`: switch `_visit_admonition`/`_depart_admonition` from the
markup-mode trailing content block (`clue_type[`) to a code-mode content-block argument
(`clue_type({ ... })`) so the body (emitted downstream as `par({...})`, `text(...)`,
`raw(...)`) evaluates instead of printing literally (D-01). Simultaneously preserve inline
markup in admonition titles by routing the title node's children through the normal inline
visitors via the existing buffer-swap idiom, replacing the `title: "{node.astext()}"` flatten
(D-02) — which also fixes the newly-discovered title double-emission bug. Strengthen the 9
existing loose unit asserts to structural checks (D-03) and add nested-content regression
coverage (D-05).

Purpose: Land the root-cause fix (D-01/D-02) with tests that would actually catch a
regression (D-03/D-05) — the loose asserts are precisely how this bug shipped.
Output: Admonition bodies + titles render as typeset prose; `tests/test_admonitions.py` green
with structural asserts.
</objective>
