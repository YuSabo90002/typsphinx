## Pattern Assignments

### `typsphinx/translator.py` — `_visit_admonition`/`_depart_admonition` (admonition, transform)

**Current (buggy) code** (lines 2253-2288):
```python
def _visit_admonition(
    self, node: nodes.Node, clue_type: str, custom_title: str = None
) -> None:
    if self.in_list_item and self.list_item_needs_separator:
        self.add_text("\n")

    title = None
    for child in node.children:
        if isinstance(child, nodes.title):
            title = child.astext()
            break

    if title:
        self.add_text(f'{clue_type}(title: "{title}")[')
    elif custom_title:
        self.add_text(f'{clue_type}(title: "{custom_title}")[')
    else:
        self.add_text(f"{clue_type}[")

def _depart_admonition(self) -> None:
    self.add_text("]\n\n")
    if self.in_list_item:
        self.list_item_needs_separator = True
```

**Root defect:** opens a **markup-mode** trailing content block (`{clue_type}[`), but everything downstream (`visit_paragraph` → `par({...})`, `visit_Text` → `text(...)`, `visit_literal` → `raw(...)`) emits **code-mode** function calls with no `#` prefix, which are inert literal text inside `[...]`. Additionally, scanning `node.children` for `title` via `.astext()` at visit-time causes a **double emission** bug: docutils' own traversal will visit the same title child again afterward through the default `visit_title`/`heading()` path, since the child is never skipped/removed.

**Verified fix** — RESEARCH.md Pattern 1 (code-mode content-block body) + Pattern 2 (deferred title via buffer-swap, reusing `visit_term`/`depart_term`):

```python
def _visit_admonition(
    self, node: nodes.Node, clue_type: str, custom_title: str = None
) -> None:
    if self.in_list_item and self.list_item_needs_separator:
        self.add_text("\n")
    self._pending_admonition_title = None  # cleared per-admonition
    self.add_text(f"{clue_type}({{")  # code-mode content-block open, NOT "["

def _depart_admonition(self, custom_title: str = None) -> None:
    self.add_text("}")
    title_expr = None
    if getattr(self, "_pending_admonition_title", None):
        title_expr = "{" + self._pending_admonition_title + "}"  # code block {…}, NOT content block [—] — buffered inline calls text(…)/emph(…) must EVALUATE (see 08.1-RESEARCH.md Pattern 2, authoritative)
    elif custom_title:
        title_expr = f'"{custom_title}"'
    if title_expr:
        self.add_text(f", title: {title_expr}")
    self.add_text(")\n\n")
    if self.in_list_item:
        self.list_item_needs_separator = True
```

Note: `custom_title` must move from `_visit_admonition`'s signature to `_depart_admonition`'s (or be stashed on `self` at visit-time) since the title argument is now built at depart-time. Every `visit_note`/`visit_important`/etc. call site (lines 2294-2340) needs `self._depart_admonition()` calls updated to pass `custom_title` through (or stash it on `self._custom_admonition_title` in `visit_*` and read it back in `_depart_admonition` with no params — pick whichever keeps the existing per-type `visit_X`/`depart_X` pattern intact).

**Verified working Typst** (RESEARCH.md Pattern 1, compiled + pypdf-extracted):
```typst
#{
  info({
    par({text("Before list.")})
    list({
      par({text("Item one.")})
    }, {
      par({text("Item two.")})
    })
  })
}
```

### `typsphinx/translator.py` — `visit_title`/`depart_title` (admonition-aware branch)

**Analog to reuse:** `visit_term`/`depart_term`, `typsphinx/translator.py:1047-1082` (buffer-swap idiom already implemented and tested indirectly via definition-list tests):

```python
# typsphinx/translator.py:1047-1082 (existing, unmodified — the pattern to copy)
def visit_term(self, node: nodes.term) -> None:
    # Start buffering term content
    self.saved_body = self.body
    self.current_term_buffer = []
    self.body = self.current_term_buffer

def depart_term(self, node: nodes.term) -> None:
    if isinstance(self.current_term_buffer, list):
        term_content = "".join(self.current_term_buffer).strip()
    else:
        term_content = ""
    if self.saved_body is not None:
        self.body = self.saved_body
    self.saved_body = None
    self.current_term_buffer = term_content
```

**Current `visit_title`/`depart_title`** (lines 176-199) — the default (non-admonition) path that must stay unchanged for section headings:
```python
def visit_title(self, node: nodes.title) -> None:
    self.add_text(f"heading(level: {self.section_level}, ")

def depart_title(self, node: nodes.title) -> None:
    self.add_text(")\n\n")
```

**New admonition-aware branch to add** (gate on `isinstance(node.parent, nodes.Admonition)` — confirmed shared mixin base for all 10 relevant node classes including generic `admonition`):
```python
def visit_title(self, node: nodes.title) -> None:
    if isinstance(node.parent, nodes.Admonition):
        self._saved_body_for_admonition_title = self.body
        self.body = []
        self._in_admonition_title = True
        return
    self.add_text(f"heading(level: {self.section_level}, ")

def depart_title(self, node: nodes.title) -> None:
    if getattr(self, "_in_admonition_title", False):
        self._pending_admonition_title = "".join(self.body)
        self.body = self._saved_body_for_admonition_title
        self._in_admonition_title = False
        return
    self.add_text(")\n\n")
```

This walks the title's children through the *normal* inline visitors (`visit_Text`, `visit_emphasis`, `visit_literal`, etc. — same escaping as body text) instead of `node.astext()`, satisfying D-02 (inline markup preserved) and closing the pre-existing unescaped-string-interpolation gap noted in RESEARCH.md's Security Domain section.

## Shared Patterns

### Buffer-swap for deferred content capture
**Source:** `typsphinx/translator.py:1047-1122` (`visit_term`/`depart_term`, `visit_definition`/`depart_definition`)
**Apply to:** `visit_title`/`depart_title`'s new admonition-aware branch. Do not invent a parallel "capture mode" flag — this exact `self.saved_body = self.body; self.body = []` / restore idiom is the established mechanism in this file.

### Unified code-mode emission (no `#`-prefix, no markup-mode toggling)
**Source:** `typsphinx/translator.py` `visit_paragraph`/`depart_paragraph` (298-337), `visit_Text` (404-429+), `visit_literal` (616-649)
**Apply to:** `_visit_admonition`/`_depart_admonition` — the fix must make the admonition body *stay in* this mode (content-block form `{clue_type}({...})`), never introduce a `#`-prefix hack at the per-visitor level (explicitly rejected as an anti-pattern in RESEARCH.md).

### String escaping for interpolated text
**Source:** `typsphinx/translator.py:424-429` (`visit_Text`) and `:636-638` (`visit_literal`) — backslash, quote, newline/tab escaping
**Apply to:** Any new string interpolation this phase introduces (e.g. `custom_title` in `_depart_admonition`). The title buffer-swap must route dynamic title text through these same visitors (i.e., through the normal walk, not a separate escape call) — this is what makes D-02 safe and is itself a security improvement (removes the current unescaped `f'title: "{node.astext()}"'` interpolation).

### Real-compile / integration test structure
**Source:** `tests/test_integration_nested_toctree.py::TestPDFCompilation` (sphinx-build subprocess + `typst.compile()` + PDF-magic-number check)
**Apply to:** `tests/test_pdf_render_gate.py` — copy the build+compile scaffolding; add the new `pypdf` text-extraction assertion layer per D-04.
