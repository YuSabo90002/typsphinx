# Phase 08.1 Plan 02: Admonition Code-Mode Rewrite Summary

**Rewrote `_visit_admonition`/`_depart_admonition` to emit gentle-clues code-mode content-blocks (`info({...})`) instead of markup-mode brackets (`info[...]`), and made admonition titles preserve inline markup via a buffer-swap instead of `node.astext()` — fixing both the literal-source-leak bug and a previously undocumented title double-emission bug.**

## Performance

- **Duration:** 6 min
- **Started:** 2026-07-11T04:28Z (approx, from prior plan's completion commit)
- **Completed:** 2026-07-11T04:34Z
- **Tasks:** 3
- **Files modified:** 2

## Accomplishments
- `_visit_admonition`/`_depart_admonition` now open/close a code-mode content-block (`clue_type({ ... })`), so every downstream body emission (`par({...})`, `text(...)`, `raw(...)`) evaluates as Typst code instead of printing as literal unevaluated source — the root cause of the Phase 7 deferred-items bug.
- `visit_title`/`depart_title` gained an admonition-aware branch (gated on `isinstance(node.parent, nodes.Admonition)`) that buffer-swaps the title's body so its children render through the normal inline visitors (`visit_Text`, `visit_emphasis`, etc.), preserving inline markup and closing the previous unescaped `f'title: "{node.astext()}"'` interpolation gap. This also fixed a discovered-in-research title double-emission bug (the old code scanned `node.children` for a title at visit-time via `.astext()`, then docutils' own traversal visited the same title again through the default heading path).
- Strengthened all 9 existing `tests/test_admonitions.py` asserts from loose substring checks (`"info[" in output`) to structural checks (`"info({" in output`, `"info[" not in output`, exact `par({text("..."))})` body forms).
- Added 3 new regression tests: `test_nested_list_in_note`, `test_nested_code_block_in_note` (D-05 nested-content coverage), and `test_admonition_title_preserves_inline_markup` (D-02 inline-markup-title + double-emission guard).

## Decisions Made
- Kept `_visit_admonition`'s `custom_title` parameter unchanged (stashed onto `self._custom_admonition_title` at visit-time, read back with no params in `_depart_admonition`) so the six existing per-type `visit_note`/`depart_note`...`visit_seealso`/`depart_seealso` call sites required zero changes.
- Introduced a dedicated `_saved_body_for_admonition_title` field rather than reusing the definition-list `saved_body` field, avoiding any state collision risk between the two buffer-swap use sites (a title could theoretically appear while a definition-list buffer-swap is also active, though no such nesting exists in current docutils semantics — the separate field removes the ambiguity entirely).
- Static custom titles ("Important", "See Also") remain plain string arguments (`title: "..."`, no escaping needed for these two hardcoded Python literals); only the dynamic, node-derived title uses the buffered code-block form (`title: { ... }`) — per RESEARCH.md's resolved Open Question 1.
- For Task 3's inline-markup-title test, used a constructed `nodes.note()` + injected title (documented in the test's docstring as not-real-rST-shape) rather than `nodes.admonition()`, since the generic `admonition` node's `visit_admonition`/`depart_admonition` handlers don't exist yet — that's Plan 03's D-06 scope.

## Deviations from Plan

None - plan executed exactly as written. The mypy type-checking gap (`self.body = self._saved_body_for_admonition_title` assigning `list[Any] | None` to a `list[Any]`-typed attribute) was resolved inline with an `is not None` guard before assignment, matching the existing pattern used by the definition-list buffer-swap (`if self.saved_body is not None: self.body = self.saved_body`) — this is not a deviation from the plan's design, just a mypy-clean implementation of the exact mechanism the plan specified.
