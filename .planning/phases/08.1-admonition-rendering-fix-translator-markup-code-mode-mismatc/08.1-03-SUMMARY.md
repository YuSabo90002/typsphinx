# Phase 08.1 Plan 03: Admonition Type Expansion (D-06) Summary

**Implemented the five previously-unimplemented Sphinx admonition types — `hint`, `error`, `danger`, `attention`, and the generic `.. admonition::` — each as a two-method visitor pair routed through Plan 02's fixed code-mode `_visit_admonition`/`_depart_admonition` choke point, using the research-verified gentle-clues 1.3.1 clue mapping (hint→tip, error→error, danger→danger, attention→warning, generic→base `clue()`).**

## Accomplishments
- Added `visit_hint`/`depart_hint`, `visit_error`/`depart_error`, `visit_danger`/`depart_danger`, `visit_attention`/`depart_attention` — each a two-line pair cloned from the existing `visit_note`/`depart_note` shape, calling `self._visit_admonition(node, "<clue_type>")` / `self._depart_admonition()`.
- Added `visit_admonition`/`depart_admonition` (generic) mapping to the base gentle-clues `clue` function — the one case where a directive-supplied title is normal/expected; it flows automatically through the Plan 02 admonition-aware `visit_title`/`depart_title` buffer-swap, requiring no `custom_title` argument and no new title-handling code.
- Added 5 new structural unit tests to `TestAdmonitionConversion`: `test_hint_converts_to_tip`, `test_error_converts_to_error`, `test_danger_converts_to_danger`, `test_attention_converts_to_warning` (each asserting the exact clue-call form and absence of the markup-mode `[` form), and `test_generic_admonition_converts_to_clue` — built on a real `nodes.admonition()` fixture with a genuine `nodes.title` child (per RESEARCH.md's Open Question 2 resolution), asserting `clue({`, the title text present exactly once, and no stray `heading(` in the output.
- All 10 admonition types (6 pre-existing from Plan 02 + these 5 new + the already-covered generic) now convert correctly through the single fixed choke point; full `tests/test_admonitions.py` suite (17 tests) passes.

## Decisions Made
- Followed the RESEARCH.md D-06 clue mapping table exactly: `hint`→`tip` and `attention`→`warning` because gentle-clues 1.3.1 has no `hint`/`attention` clue (verified against the cached `theme.typ` dict — calling a non-existent clue would `KeyError` at compile); `error` and `danger` map 1:1 to their own predefined clues; the generic `.. admonition::` maps to the base `clue()` function (unstyled), not `info`, since it's the one type where a real directive-supplied title is normal.
- The generic admonition test uses a real `nodes.admonition()` fixture with a genuine `nodes.title` child, per RESEARCH.md Open Question 2's resolution — not `nodes.note()` with an injected title (which is not how docutils actually produces custom-title admonitions), distinguishing it from the existing `test_admonition_with_title_in_content` test that intentionally uses the non-realistic `nodes.note()`-with-title shape to exercise the buffer-swap mechanism in isolation.
- Split the combined implementation into two atomic per-task commits (rather than one combined commit) by temporarily removing Task 2's code/test, verifying and committing Task 1 in isolation, then restoring and verifying/committing Task 2 — preserving the plan's task-level commit granularity.
