<objective>
Pin the three runtime dependencies (`typst`, `sphinx`, `docutils`) to a reproducible, known-good, mutually-compatible combination by adding upper bounds (floors unchanged) in `pyproject.toml`, mirroring those ceilings in `tox.ini` for documentation-truth, removing the dead `sphinx-testing` dependency, regenerating and committing `uv.lock`, then empirically confirming the `docs-pdf` target builds a PDF (no `kai` error) and recording the confirmed typst patch + the ceiling load-bearing finding in `PROJECT.md`.

This is the actual bug fix. It lands as its own commit stream so a red/green CI result is unambiguous. The `black` reformat (LINT-01/02) is deliberately NOT in this plan — it lands separately in plan 01-02 per D-04.

Purpose: Loose, unbounded ranges let a fresh resolution jump to `typst==0.15.0`, which breaks PDF compilation on `mitex 0.2.4`'s `kai` symbol (turned into a hard error by typst 0.15). Pinning the compiler back to the 0.14.x line where the four bundled `@preview` packages compile is the root-cause fix.
Output: Edited `pyproject.toml` + `tox.ini`, regenerated `uv.lock`, updated `PROJECT.md` Key Decisions table.
</objective>
