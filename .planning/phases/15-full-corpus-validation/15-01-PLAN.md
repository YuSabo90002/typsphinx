<objective>
Create the standing GATE-02 corpus render gate: a NEW `slow`-marked pytest module
`tests/test_corpus_gate.py` that shallow-clones Sphinx's own `doc/` tree at the tag
matching the installed `sphinx.__version__` (D-01), augments the real `doc/conf.py`
in-place with a 2-line typsphinx wiring append (D-03), builds the FULL tree (D-02)
through the `typstpdf` builder, and asserts the compiled PDF exists / is non-empty /
begins with `%PDF` with no fatal `TypstCompilationError` (SC#1). The same build's
stderr feeds a MULTILINE-anchored `unknown_visit` frequency parser (SC#2 catalogue
raw material). The test `pytest.skip`s on no-network/clone-failure (D-05) and is
excluded from the fast/CI suite by the existing `slow` marker (D-04).

end-to-end through typstpdf with no fatal error." This plan delivers that gate plus
the SC#2 catalogue parser it depends on.
Output: `tests/test_corpus_gate.py` (module scaffolding + clone/cache helper +
conf-augmentation helper + parameterized subprocess helper + `catalogue_unknown_visit`
parser + a fast unit test for the parser + the `TestCorpusRenderGate` slow class).
</objective>

| github.com repo → local cached clone | untrusted-origin source tree (Sphinx official repo, pinned tag) is fetched to disk |
| cloned `conf.py` → `sphinx-build` subprocess | arbitrary Python in the cloned conf.py is executed by the build |
| tag / path strings → git & sphinx subprocess args | test-constructed strings become subprocess arguments |
