### 4. Triage CR-01 — `writer.py` template-import path malformed for a docname containing a literal `_template` segment

expected: A human decides whether this must be fixed before Phase 22.1 / v0.6.2 closes, or is
acceptable to file as a tracked follow-up backlog item / phase.

`TypstWriter.translate()` (`typsphinx/writer.py:163-165`) computes the template import path by reusing
`TypstTranslator._compute_relative_include_path("_template", docname)`. When a master's docname
contains a directory segment literally named `_template` (e.g. docname `_template/index`), the
relativization logic mistakes the fake `"_template"` sentinel target for a real shared path component
and returns `"."`, which becomes a malformed import (`"..typ"` / `"../.typ"`) instead of a reference to
the real `outdir/_template.typ`.

Reproduced independently **twice**: once by the code reviewer against the installed package, and again
by the verifier directly against `typsphinx.translator.TypstTranslator._compute_relative_include_path`
(`fn(Dummy(), "_template", "_template/index") == "."`). It is not a false positive.

Why it is not a mechanical phase failure: no ROADMAP success criterion, plan must-have, or fixture in
this phase uses a docname containing `_template` — the gate fixture uses `api/index`. The bug is
nonetheless in code **this phase introduced**, is untested, and is tracked by no backlog item.

why_human: Whether an untriaged critical-severity code-review finding, in code this phase introduced,
blocks phase closure is a policy decision, not a mechanically verifiable fact.
