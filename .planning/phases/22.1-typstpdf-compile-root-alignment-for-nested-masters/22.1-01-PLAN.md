<objective>
Align the `-b typstpdf` compile basis with the translator's docname-relative emission basis by
compiling each master's own `.typ` file at its real on-disk location, and replace the silent
compile-failure swallow with attempt-all-then-raise.

Purpose: `TypstPDFBuilder.finish()` currently reads a master's `.typ` into a string and hands it to
`compile_typst_to_pdf()`, which writes it to a `NamedTemporaryFile(dir=self.outdir)` at the **outdir
root** and compiles that copy. Typst resolves relative paths against the file being compiled, while
the translator emits them docname-relative — so any master not sitting at the outdir root already
fails today, and the bare `except Exception: logger.error(...)` hides that failure behind exit code 0.
D-01 removes the temp copy entirely so the divergence becomes structurally impossible; D-04/D-05 make
the failure un-ignorable.

Output: a new file-path compile entry point in `pdf.py`, a rewired and fail-loud `finish()` loop in
`builder.py`, and the repair of the two `tests/test_pdf_generation.py` cases that encode the old
internals plus a new unit test pinning the path-resolution invariant.
</objective>

<assumption_delta_decision>
## Assumption delta — the compile basis becomes per-master

**Noun that is now primary:** *the master's own directory* (`outdir/<docname-dir>/<stem>.typ`).
The *outdir root* is demoted: it is no longer the location where compilation happens, and survives
only as `root_dir` — the path-escape boundary handed to `typst.compile(root=...)`.

**Decision: `promote`.**

Rationale: D-01 deletes the temporary copy outright, so after this plan there is exactly one notion
of "where the compile happens" — the file the write phase already produced. Keeping a second,
outdir-root-based notion alongside it (the `add-alongside` option, and the ROADMAP's originally
stated minimal fix of *relocating* the temp file) would preserve two divergable bases indefinitely;
the whole point of PDF-02 is that the divergence must become structurally impossible rather than
merely corrected for today's fixture.

**Contract change this promotion carries with it:** the multi-master `finish()` loop stops being
"one global compile at one global root, failures logged and forgotten" and becomes "N per-master
compiles, each with its own basis, failures accumulated into one raised report" (D-04/D-05). The
loop's failure semantics are therefore also promoted from swallow-and-continue to
attempt-all-then-raise.

**Accepted debt:** the legacy `compile_typst_to_pdf(content, root_dir)` still holds the demoted
outdir-root basis internally. That is deliberate (D-02: 18 existing call sites stay untouched) and
time-boxed — D-03 announces its removal for v0.7. What would force finishing the promotion earlier:
any *new* production call site adopting the content-string form, or a user bug report traced to its
documented root-relative limitation.
</assumption_delta_decision>

<flagged_assumptions>
## Flagged assumptions (edge-coverage probe, unresolved — not auto-resolved, not dropped)

The spec-less edge probe surfaced exactly **one** applicable edge for PDF-02 and left it
`unresolved`. It is recorded here as an explicit flagged assumption rather than silently backstopped.

**FA-01 — concurrency / interruption (`category: concurrency`, probe: "If interrupted or run in
parallel, what is guaranteed?")**

Position taken by this plan, stated so a reviewer can disagree with something concrete:

1. **Multi-master sequencing IS guaranteed and IS tested.** Masters are compiled sequentially in
   `typst_documents` order. A failure of master #2 of 3 does not prevent #3 from being attempted or
   its `.pdf` from being written; exactly one exception is raised after the loop, naming every
   failure. This half is covered by Task 3's rewritten failure-contract test and is a `truths` entry,
   not an assumption.
2. **Interruption gives no atomicity guarantee, and this plan does not add one.** `.pdf` files are
   written with a plain `open(..., "wb")`, so a SIGINT mid-write can leave a truncated `.pdf` in
   `outdir`. Nothing detects that on the next build. This is pre-existing behavior, unchanged here.
   What *does* improve: the pre-fix code could also leave an orphaned `tmpXXXX.typ` in `outdir` on
   interruption, and D-01 removes that failure mode entirely from the `finish()` path.
3. **Two concurrent builds sharing one `outdir` are not supported and no locking is added.** Sphinx
   itself makes no such guarantee for a shared output directory. The change narrows rather than
   widens the collision surface (no temp filename to collide on), but the `.pdf` write is still
   last-writer-wins.

**Why this is left unresolved rather than closed:** closing it would mean adding atomic
write-then-rename and/or an outdir lock, which is new behavior nobody asked for in a bug-fix release
and would touch the output-layout question that CONTEXT.md's `<deferred>` section explicitly dropped
on 2026-07-21. Recorded for a future milestone; **not** silently dismissed.
</flagged_assumptions>
