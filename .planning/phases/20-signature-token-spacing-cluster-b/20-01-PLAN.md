<objective>
Restore intra-signature token spacing for FID-07 (the `class `/`exception ` annotation
prefix) and FID-08 (all C/C++ `desc_signature` and inline `c/cpp:expr` inter-token spaces) in
one fix, by reducing `visit_desc_sig_space`/`depart_desc_sig_space` to `pass`/`pass` — deleting
the buggy `self.body.append(" ")` + `raise nodes.SkipNode` short-circuit that discards the
node's own `Text(" ")` child before the already concat-context-aware `visit_Text` dispatch can
see it.

Purpose: A `desc_sig_space` currently writes raw Typst *source* whitespace between two code-mode
statements, which Typst auto-joins with zero rendered space (the token-level instance of Phase
19's proven "code-mode whitespace is cosmetic-only" invariant). Letting the child `Text(" ")`
stream through `visit_Text` emits a real content-space VALUE that renders. The four sibling
`desc_sig_*` handlers are already `pass`/`pass`; this makes `desc_sig_space` match them.
Output: The single-choke-point source fix plus a new real-compile GATE-01 fixture with
structural `.typ`, `%PDF`, and pypdf extracted-text adjacency asserts (D-07).
</objective>
