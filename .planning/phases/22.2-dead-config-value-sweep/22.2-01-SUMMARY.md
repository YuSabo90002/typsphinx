# Phase 22.2 Plan 01: Dead Config-Value Registration Removal Summary

**Deleted the `typst_output_dir` and `typst_author_params` Sphinx config registrations and purged `typst_output_dir` from every forward-looking doc/example/test surface, with no deprecation period.**

- **Completed:** 2026-07-21

## Accomplishments
- `typsphinx/__init__.py setup()` no longer registers `typst_output_dir` or `typst_author_params`; the remaining 12 `add_config_value` calls (including the bracketing `typst_authors`/`typst_debug`) are untouched
- `docs/configuration.rst`'s "Output Configuration" section and its composite full-config example line are gone; `examples/advanced/conf.py`/`README.md` no longer carry the commented-out output-directory blocks
- Both registration-only tests (`test_typst_output_dir_config_registered`, `test_typst_output_dir_default_value`) deleted outright — no replacement negative test, per plan instruction
- `tests/test_documentation_configuration.py`'s `required_configs` list matches the trimmed docs
- Phase 23's ROADMAP success criterion now stages a `### Removed` CHANGELOG note naming BOTH `typst_output_dir` and `typst_author_params`, closing the hand-off gap CONTEXT.md flagged

## Decisions Made
- No deprecation shim, warning, or `Config.add_deprecated` call added for either removed config — per D-13, Sphinx silently ignores unregistered `conf.py` variables, making removal behaviorally invisible to existing users; a deprecation period would contradict the owner's 2026-07-21 decision.
- No replacement negative test written for the deleted registration-only tests — CONF-01's registration-absence is already proven structurally by the `__init__.py` edit; a test asserting the attribute's absence would just be the same registration-only assertion inverted.
