<objective>
Delete the two dead Sphinx config registrations (`typst_output_dir`, the legacy author-params
config) and purge the first one from every forward-looking user-facing surface, per D-13 and the
registration half of D-07.

Purpose: CONF-01. Both configs are registered and (for one of them) documented, but are read
nowhere in `typsphinx/`. The existing tests only assert that the values are *registered* — they
stay green while the feature is dead. Removing the registration is what actually makes the escape
impossible to re-open.

Output: a `typsphinx/__init__.py` whose config registrations all correspond to a live reader; a
`docs/configuration.rst` with no "Output Configuration" section; two deleted registration-only
tests; a `required_configs` list that matches the docs; and a ROADMAP Phase 23 criterion that names
both removals so the `[0.6.2]` CHANGELOG carries a correct `### Removed` section.
</objective>

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| developer `conf.py` → typsphinx config read | `conf.py` is arbitrary Python executed by Sphinx; it is already fully trusted before typsphinx runs. Removing a registration cannot expand this boundary. |
| repository docs → user expectation | Docs are the only contract a user has about which config names exist; a doc that names a nonexistent config is a (benign) integrity defect, which this plan closes. |
