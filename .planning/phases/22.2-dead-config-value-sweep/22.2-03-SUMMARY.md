# Phase 22.2 Plan 03: TemplateEngine Parameter Pipeline Repair Summary

**Repaired `TemplateEngine.map_parameters()`/`render()` to stop silently injecting defaults on the Typst-Universe package path, hoist essential `@preview` imports to every master unconditionally, invert parameter precedence so explicit config wins, wire `typst_authors` as a native array of dictionaries, and delete the dead `typst_author_params` legacy surface.**

## Performance

- **Duration:** ~35 min
- **Started:** 2026-07-21T21:13Z (approx, per STATE.md `last_updated`)
- **Completed:** 2026-07-21T21:25:29Z
- **Tasks:** 3
- **Files modified:** 4 (`typsphinx/template_engine.py`, `typsphinx/writer.py`, `typsphinx/builder.py`, `tests/test_template_engine.py`)

## Accomplishments

- Every master document's emitted `.typ` now carries the four essential `@preview` imports (codly, codly-languages, mitex, gentle-clues) plus the codly initialization exactly once, regardless of whether a template file or a Typst Universe package is configured — closing BUG-F where package-only masters previously got none of them.
- A `typst_package`-configured `TemplateEngine` with no explicit `parameter_mapping` now returns an empty parameter dict from `map_parameters()` — no more silent `title`/`authors`/`date` injection into a package function that never asked for them (BUG-B). Gated at both injection sites (constructor default AND the fill-if-missing back-fill), per the plan's LOAD-BEARING CORRECTION.
- `render()`'s parameter merge order is inverted: auto-derived Sphinx metadata is now the fallback, and explicit `typst_template_function["params"]` values are authoritative on a key collision (BUG-E).
- `typst_authors`, when set, now flows through `map_parameters()` as a native `list[dict]` that the existing generic Typst value formatter serializes as a proper array of dictionaries — never as a quoted string (BUG-C).
- The legacy `typst_author_params` configuration surface — dead since introduction — is deleted everywhere: the `TemplateEngine` constructor keyword and docstring line, the `self.typst_author_params` attribute, the `elif` branch inside `_format_authors_with_details()` (and its now-unused `authors` parameter), both `TemplateEngine(...)` construction sites in `writer.py` and `builder.py`, and the sole backward-compatibility unit test that exercised it.
