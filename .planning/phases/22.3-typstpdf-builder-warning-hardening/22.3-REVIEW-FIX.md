# Phase 22.3 — Code Review Fix Record

All three findings from `22.3-REVIEW.md` were fixed inside the phase, following the
Phase 22.1 precedent where that review's Critical was closed in-phase as gap `G-22.1-4`
(owner decision, 2026-07-22).

## CR-01 (Critical) — RESOLVED

**Finding:** `TypstWriter._is_master_document()` (`typsphinx/writer.py:56`) indexed
`doc_tuple[0]` with no guard. `_is_master_document()` runs during `write_doc()` for
*every* document, so a malformed (empty-tuple) `typst_documents` entry raised
`IndexError: tuple index out of range` in the **write** phase — aborting `sphinx-build`
with a raw traceback before `TypstPDFBuilder.finish()` (the code WR-01 hardened) ever ran.
No PDF was produced for any master, including a valid one.

**Independently reproduced before fixing** (orchestrator, two-document project with
`typst_documents = [("index",…), ("ghost",…), ()]`):

```
File "typsphinx/writer.py", line 56, in _is_master_document
    if doc_tuple[0] == docname:
IndexError: tuple index out of range
SPHINX_EXIT=2 — PDFs produced: (none)
```

This falsified the phase's own D-02 attempt-all-then-raise claim ("masters that compile
successfully still get their `.pdf` written") for any project with more than one document.
The new gate passed only by accident of fixture shape: the fixture was a single-document
project with the valid master listed first, so the scan returned on iteration 1 and never
reached the malformed entry.

**Fix (2 parts — the guard alone would have left the gate unable to detect a regression):**

1. `typsphinx/writer.py` — the scan now skips falsy entries
   (`if doc_tuple and doc_tuple[0] == docname:`), matching the guards
   `_compute_master_included_docnames()` and `_resolve_output_stem()` already use in
   `builder.py`. Malformed entries never match here; reporting them stays `finish()`'s sole
   job, so D-06's scope fence and D-07's `repr(doc_tuple)` identifier are both untouched.
2. `tests/fixtures/missing_and_malformed_master_gate/` — added `chapter1.rst`, a second
   **non-master** document reached from `index.rst`'s toctree and deliberately absent from
   `typst_documents`. Writing it forces the scan to traverse the whole list without matching,
   so it reaches the malformed `()` entry. `tests/test_missing_and_malformed_master_gate.py`
   gained two assertions: `"IndexError" not in result.stderr` and
   `(build_dir / "chapter1.typ").exists()`.

**Pre-fix must-fail proof (same SC#5 discipline as the rest of the phase).** The guard was
temporarily reverted and the gate re-run: **2 failed** — the build aborted in the write phase
and neither test's assertions held. The file was then restored byte-identically
(`md5sum 858aa4fe373a2dfc5b03cb1f3289f632`) and the gate re-run: **2 passed**.

`"IndexError"` is a Python builtin exception name, not an upstream `typst-py` diagnostic
string, so asserting its absence does not reintroduce the WR-02 coupling this phase removed.

## WR-01 (Warning) — RESOLVED

**Finding:** `tests/test_nested_master_render_gate.py` still asserted
`"file not found" not in result.stderr` — a negative assertion on `typst-py`'s own
uncontracted diagnostic wording. It is the "always passes, never red" failure mode: an
upstream rewording would silently make it vacuous. That is the same defect class WR-02
removed from the rest of this module, so leaving it half-closed WR-02.

**Fix:** assertion removed, replaced by a comment recording *why* no such assertion belongs
there. `returncode == 0` already proves the build succeeded, and the adjacent
`"Typst compilation failed" not in result.stderr` check is a typsphinx-authored string this
project owns.

This supersedes the planner's `<flagged_assumptions>` entry that proposed leaving it in place.

## IN-01 (Info) — RESOLVED

**Finding:** `test_builder_appends_failure_for_unknown_docname` asserted the
`not in found_docs` precondition for `"ghost"` but not for the non-ASCII docname
`日本語ドキュメント`, so that half could silently start exercising D-04's fallback branch
instead of the `found_docs`-discriminating branch it is written to cover.

**Fix:** added the symmetric precondition assertion with the rationale inline.
