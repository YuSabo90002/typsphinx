<objective>
Fix FIG-02 (Issue #114's second fatal bug) inside `typsphinx/translator.py`: replace the
`node.astext()` caption handling in `visit_caption`/`depart_caption` with the proven admonition-title
buffer-swap idiom so a figure caption renders through the normal visitor chain into the figure's
`caption:` named argument, and add a `refid` fallback branch to `visit_reference` so an internal
same-document `:target:` link compiles as well as an external-URL one. This is the phase's
highest-priority and most bug-prone fix (state-machine surgery on load-bearing visitor state), so it
gets its own isolated plan.

Purpose: today `depart_caption` re-derives the caption via `astext()` while the caption's Text
children ALSO stream into the live body — producing both a syntax-breaking `image(...)text(...)`
juxtaposition and a doubled caption. The buffer-swap eliminates both at the root and, as a free side
effect, routes markup-special caption characters through `visit_Text`'s existing string-literal
escaping.

Output: `_saved_body_for_figure_caption` state var; buffer-swapped `visit_caption`/`depart_caption`;
code-block caption emission in `depart_figure`; the `refid` branch in `visit_reference`.

**Requirements coverage:** FIG-02 (per D-03).

`typsphinx/translator.py` (Plan 11-01 also edits it) — there is no logical dependency on 11-01's code;
this ordering only prevents concurrent same-file edits across parallel agents.
</objective>
