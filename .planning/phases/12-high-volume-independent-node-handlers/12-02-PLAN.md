<objective>
Land XREF-01: prove same-document `refid` cross-references (`:ref:` section anchors and `:term:`
glossary links) render as working PDF links, and fix the required companion bug that makes them
possible. Research (12-RESEARCH.md Part 2) proved via a live `typst.compile()` that the existing
`visit_reference` refid branch already emits `link(<term-Widget>, …)` correctly for a `:term:` xref —
but `visit_term`/`depart_term` never emits a Typst `<label>` anchor for the term's docutils id, so
`<term-Widget>` is undefined and Typst aborts the ENTIRE PDF with `label <term-Widget> does not exist`.
This is a currently-live fatal bug, not optional polish: the phase's own mandated `:term:` GATE
fixture (D-04) cannot pass without the fix.

Purpose: this is the FATAL companion fix flagged as the phase's primary risk. It reuses Phase 11's
already-proven bracket-wrap markup-mode label-attachment pattern (`[#{...} <label>]`, the same idiom
`visit_title`/`visit_figure` use). Per D-03 the refid link branch itself is confirm-and-cover — do NOT
add parallel machinery; the only code change is emitting the missing anchor in `depart_term`.

Output: the `depart_term` label-anchor fix, one fixture project exercising a `:ref:` section anchor +
a `:term:` glossary ref, and one real-compile test class that MUST fail until the anchor fix lands.

**Requirements coverage:** XREF-01 (per D-03 confirm-and-cover + D-04 safe-side/never-`link("", …)`).
GATE-01 standing bar extended (SC#5).

**Wave note:** `depends_on: [12-01]` is a file-ownership sequencing dependency (shared
`translator.py` + `test_pdf_render_gate.py`), not a logical one — XREF-01 is independent of VER-01.
</objective>
