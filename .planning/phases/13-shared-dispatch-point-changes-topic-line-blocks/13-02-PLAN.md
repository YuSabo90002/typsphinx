<objective>
Add `line`/`line_block` support to `typsphinx/translator.py` so line-block content (addresses,
epigraph shapes, poetry stanzas) renders with every line break preserved via a real `linebreak()`,
and nested line blocks reproduce their structural indentation via a per-depth `h()` spacer — both
compile-safe with zero markup-mode involvement (D-03/D-04).

Purpose: BLK-03 — today `line`/`line_block` fall through `unknown_visit` and their text children run
together with no separator at all (RESEARCH.md Pitfall 3 baseline). This plan gives them verbatim
breaks and depth-tracked indentation.

Output: four new visitor methods (`visit_line_block`/`depart_line_block`/`visit_line`/`depart_line`),
one new integer instance flag (`self._line_block_depth`), and dedicated unit tests in
`tests/test_line_blocks.py`. Independent of the `visit_title` generalization (13-01) — sequenced after
it only because both edit `translator.py` (merge-conflict avoidance). The GATE-01 real-compile proof
(address + poem sentinels as separate extracted lines) lives in the Wave-3 combined render gate
(13-03).
</objective>
